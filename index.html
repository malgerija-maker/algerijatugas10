01 LATIHAN 1: CONNECTION POOLING
Dartpad

Menghemat alokasi memori melalui pola **Database Connection Pooling** menggunakan Factory Constructor. Jumlah koneksi aktif dibatasi maksimal 3 koneksi, dan koneksi yang telah dilepas (*idle*) akan secara otomatis dimanfaatkan kembali. — Tema gunung mengilhami ketahanan dan efisiensi layaknya aliran sungai di pegunungan.

Kesimpulan Latihan 1: Factory constructor pada koneksi database mampu mengelola resource secara optimal dengan reuse objek idle, mencegah overload dan pemborosan memori. Mekanisme pool membatasi jumlah koneksi aktif, memastikan stabilitas aplikasi seperti sistem alami pegunungan yang menjaga keseimbangan.
FILE: libconnection_factory.dart
class ConnectionFactory {
  final String id;
  final String dbName;
  final String host;
  final int port;
  bool isReleased = true;

  static const int maxConnections = 3;
  static final Map> _poolRegistry = {};

  ConnectionFactory._(this.id, this.dbName, this.host, this.port);

  factory ConnectionFactory(String dbName, {String host = 'localhost', int port = 5432}) {
    final key = '${dbName.toLowerCase()}@$host:$port';
    if (!_poolRegistry.containsKey(key)) _poolRegistry[key] = [];
    final connList = _poolRegistry[key]!;
    for (var conn in connList) {
      if (conn.isReleased) {
        conn.isReleased = false;
        print('🏔️ [Pool Hit] Reusing idle connection: ${conn.id} for $key');
        return conn;
      }
    }
    if (connList.length < maxConnections) {
      final connIdentifier = 'conn_${dbName.toLowerCase()}_${connList.length + 1}';
      final newConnection = ConnectionFactory._(connIdentifier, dbName, host, port);
      newConnection.isReleased = false;
      connList.add(newConnection);
      print('🌲 [Pool Miss] Created new connection: $connIdentifier untuk $key');
      return newConnection;
    }
    throw StateError('⛰️ Batas maksimum koneksi database tercapai ($maxConnections)');
  }
  void query(String sql) { /* ... */ }
  static void release(ConnectionFactory conn) { /* ... */ }
}

import 'libconnection_factory.dart';
void main() {
  print('📌 DEMO POOL KONEKSI - ALGERIJA 📌');
  final dbConn1 = ConnectionFactory('testdb');
  dbConn1.query('SELECT * FROM users');
  final dbConn2 = ConnectionFactory('testdb');
  dbConn2.query('SELECT * FROM products');
  ConnectionFactory.release(dbConn1);
  final dbConn3 = ConnectionFactory('testdb');
  dbConn3.query('SELECT * FROM logs');
  print('Apakah reuse? \${identical(dbConn1, dbConn3)}');
}

02 LATIHAN 2: PLATFORM NOTIFICATION SELECTOR
Dartpad

Menerapkan **Subclass Selection** melalui Factory Constructor. Klien memanggil satu interface terpadu, sedangkan factory constructor secara dinamis mengembalikan instansi subclass yang spesifik (FCM, APNS, WebPush, WNS) sesuai platform target. Setiap platform mendapatkan implementasi notifikasi yang tepat tanpa mengubah kode pemanggil.

03 LATIHAN 3: SHAPE CACHING (FLYWEIGHT)
Dartpad

Menerapkan **Flyweight Pattern** melalui caching bentuk. Apabila dipanggil instansi dengan tipe, ukuran, dan warna yang sama persis, sistem akan mengambil dari cache. Seperti formasi batuan yang khas di pegunungan.

Kesimpulan Latihan 3: Dengan memanfaatkan factory constructor dan cache statis, objek yang identik tidak diciptakan berulang kali. Pola flyweight menghemat memori secara signifikan saat banyak objek serupa digunakan, meningkatkan performa aplikasi. Cache key yang unik (type+size+color) menjamin reuse tepat sasaran.

🏁 KESIMPULAN AKHIR PRAKTIKUM
Refleksi

Berdasarkan implementasi keempat latihan (Connection Pooling, Notification Selector, Shape Caching, Animal Factory) dengan tema GUNUNG, dapat ditarik beberapa kesimpulan utama mengenai Factory Constructor di bahasa pemrograman Dart:

📌 Ringkasan Materi Factory Constructor:
1. Pooling & Resource Management: Factory memungkinkan pengelolaan instance terbatas (connection pool) dengan memeriksa objek idle sebelum membuat yang baru, mencegah resource leak.
2. Subclass Selection Dinamis: Factory dapat mengembalikan berbagai tipe turunan berdasarkan parameter runtime tanpa mengubah kode klien (polimorfisme alami).
3. Flyweight / Caching: Dengan cache statis internal, factory menghindari duplikasi objek identik, menghemat memori dan mempercepat eksekusi.
4. Validasi Terpusat: Semua logika validasi (umur negatif, nama kosong, dll) dapat dipusatkan di factory, memastikan objek yang terbentuk selalu dalam keadaan valid.
5. Fleksibilitas & Maintainability: Perubahan cara pembuatan objek cukup di dalam factory tanpa mengganggu seluruh basis kode yang menggunakan factory tersebut.

🏔️ Kesimpulan Akhir: Factory Constructor di Dart adalah pattern yang sangat esensial untuk membangun sistem yang robust, hemat memori, mudah dikembangkan, dan sesuai dengan prinsip OOP (Open/Closed, Single Responsibility). Dengan tema gunung, kita belajar bahwa seperti ekosistem pegunungan yang efisien, factory constructor mengatur "siklus hidup" objek secara optimal.
